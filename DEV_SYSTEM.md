## What changed (and why)

Claude Code's **auto memory** (v2.1.59+) now accumulates learnings across sessions on its own —
debugging insights, architecture notes, workflow habits — the things the old `dreams.md` §3 and the
session ledger were hand-maintaining. So the system collapses to fewer, leaner pieces:

| Old (five-role) | New (auto-memory era) |
|---|---|
| `CLAUDE.md` (everything) | `CLAUDE.md` (trim <200 lines — always-true rules only) + **`.claude/rules/`** (per-topic, conditionally loaded) |
| `dreams.md` §1 + §2 + §3 | `dreams.md` **§1 only** (active target + milestone checklist) |
| hook gate **+** appends incidents to dreams §2 | hook gate **only** (no dreams writing) |
| `session_ledger.md` | **deleted** — `git log` is the history |
| gotchas/learnings written by hand | **auto memory** captures them |

### The dividing line (the principle to keep)

- **Committed + deterministic → `CLAUDE.md` / `.claude/rules/`.** Must-follow rules and load-bearing
  gotchas. Versioned with the code, shared, guaranteed to load.
- **Local + probabilistic → auto memory.** Incidental learnings and session history. Not in git,
  not shared, captured at Claude's discretion.

A critical rule must never quietly live only in auto memory — if it must be followed, it goes in a
committed file.

## Placement rule (repo vs suite root)

> Anything a coding agent reads while working *inside* a repo lives in that repo; cross-project
> knowledge/strategy lives in the **suite root** — `Documents/Projects/boojy/`, the container folder
> that holds all the repo subfolders (moved here from the vault on 2026-06-03).

- **In each repo:** `CLAUDE.md`, `.claude/rules/`, `dreams.md` (§1), `.claude/` (settings, hook,
  skill), `README.md`, `CHANGELOG.md` (only if it already ships releases), optional `FEATURES.md`.
- **In the suite root (`Documents/Projects/boojy/`):** the Vision doc, brand docs, and this file.
  Retired docs live in `Documents/Projects/boojy/archive/`. (Operational status —
  `SUITE_STATUS.md` — lives in the vault, not here; see the header note.)

`CLAUDE.md` / `.claude/rules` / `dreams.md` cannot move out of their repo — Claude Code reads them
from the repo and the hook uses repo-relative paths.

# Part A — how each piece works

- **`CLAUDE.md`** — authoritative rules: architecture, commands, shipping workflow, *always-true*
  conventions. Keep under ~200 lines. Read every session. The Memory-Sync line points only at
  `dreams.md` §1 now (no ledger, no §2/§3).
- **`.claude/rules/`** — one topic per file. A file with no frontmatter loads every session
  (global). A file with `paths:` frontmatter is *meant* to load only when working on matching files:
  ```
  ---
  paths:
    - "src/editor/**/*.ts"
  ---
  ```
  ⚠️ **Caveat (early-2026 bugs):** conditional loading is unreliable — `paths:` rules currently load
  globally regardless of the working dir (#16299), only inject on *read* not *write* of a matching
  file (#23478), and user-level `~/.claude/rules/` `paths:` are ignored (#21858). **So treat
  `.claude/rules/` as organization, not context savings, for now**, and keep genuinely global rules
  in `CLAUDE.md`. The structure is the documented direction; the savings arrive when the bugs do.
- **`dreams.md` (§1 only)** — the Active Engineering Target + a milestone checklist (`- [ ]` /
  `- [x]`). Volatile working state. That's it — incident logs and the backlog/scratchpad are gone.
- **`.claude/hooks/post-edit-validation.sh`** — the gate, unchanged in spirit: on a source edit it
  runs the repo's validation commands, with graceful-skip + `mktemp`. **It no longer writes to
  `dreams.md`.** On failure it must still **print the error to stderr and exit non-zero** (don't let
  removing the logging silently swallow failures). Delete the old `log_failure_to_dreams` function.
- **`.claude/settings.json`** — `compactPrompt` (preserve dreams §1 tasks + CLAUDE.md/rules +
  file paths + error signatures; drop chatter) + two PostToolUse hooks: the `Bash` docs-nudge and
  the `Edit|Write|MultiEdit` gate.
- **`session-metrics` skill** — kept (git line-velocity). The session *ledger* is gone.
- **auto memory** — owns learnings + history. Skim `/memory` after a big refactor to see what it
  captured (this replaces the old hand-pruning ritual).
- **`README.md` / `FEATURES.md` / `CHANGELOG.md`** — outward-facing, unchanged. Don't add a
  CHANGELOG until an app has users; but **keep existing changelogs that already track real releases**
  (notes, design).

# Part B — setup / migration playbook

### Target skeleton

```
<repo>/
├── CLAUDE.md                 # <200 lines, authoritative rules; Memory-Sync → dreams.md §1
├── dreams.md                 # §1 only (active target + milestone checklist)
├── README.md
├── CHANGELOG.md              # only if it ships releases
├── FEATURES.md               # optional — user-facing apps only
└── .claude/
    ├── settings.json         # compactPrompt + 2 PostToolUse hooks
    ├── settings.local.json   # GITIGNORED
    ├── rules/                # one topic per file; paths: frontmatter where file-specific
    ├── hooks/post-edit-validation.sh   # gate only (no dreams writing)
    └── skills/session-metrics/SKILL.md
```

### The one variable: the gate

| Repo | Stack | Hook gate (per source edit) |
|------|-------|-----------------------------|
| boojy-design | Vite/React/TS + Vitest | `biome check --write` → `pnpm typecheck` (.ts/.tsx) → `vitest related` |
| boojy-notes | Vite/React/TS + Vitest | same trio |
| boojy.org | Astro static | `astro check` (scoped to `website/`, skip until toolchain ready); Biome lint separate |
| Boojy Audio | Flutter + Rust | `.dart` → `flutter analyze`; `.rs` → `cargo check`/`clippy` (in `engine/`). Full `cargo test`/`flutter test` stay manual/CI, not in the per-edit loop |
| boojy-cloud *(TODO)* | Supabase edge fns | _suggested:_ minimal — `deno check`/`deno lint`, else no gate |

### Migration steps (apply on a branch; plan + diffs before deleting; green gates before commit; never commit to the default branch; skip steps for files a repo doesn't have)

1. **`CLAUDE.md`** — trim to <200 lines of always-true rules. Move per-area gotchas to
   `.claude/rules/` (next step). Update the Memory-Sync line to point only at `dreams.md` §1.
2. **`.claude/rules/`** — create it; split gotchas one-topic-per-file. File-specific ones get
   `paths:` frontmatter; genuinely global ones stay in `CLAUDE.md` or a no-`paths` rule file.
3. **`dreams.md`** — reduce to §1 only. Migrate still-relevant §3 gotchas into `.claude/rules/`;
   drop the rest. Delete §2 and §3.
4. **hook** — keep the gate + graceful-skip + `mktemp`; remove the dreams-§2 append and the
   `log_failure_to_dreams` function; ensure failures still print to stderr + exit non-zero. Update
   `settings.json` only if the hook path changed (it shouldn't).
5. **`session_ledger.md`** — delete it. Remove ledger references from `CLAUDE.md`.
6. **CHANGELOG** — don't add until the app has users; keep existing ones that track real releases.
7. **Verify** — trigger the hook on a test edit (still gates correctly), run `/memory` (CLAUDE.md +
   rules load, auto memory on), and note what auto memory already captured so you can see what's now
   redundant.

### One-time user-level fix (not per-repo)

`/memory` showed both `~/CLAUDE.md` and `~/.claude/CLAUDE.md` loading. Merge `~/CLAUDE.md` into
`~/.claude/CLAUDE.md`, then delete `~/CLAUDE.md`. `~/.claude/CLAUDE.md` is the official user file that
loads for every project; `~/CLAUDE.md` only loads by directory-walk accident.
